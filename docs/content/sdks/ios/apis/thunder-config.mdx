# `ThunderConfig`

`ThunderConfig` is the configuration struct for the Thunder Swift SDK. Pass an instance to [`ThunderClient.initialize(config:)`](/sdks/ios/apis/thunder-client#initialize) to configure the SDK.

## Usage

```swift title="MyApp.swift" showLineNumbers
let config = ThunderConfig(
    baseUrl: "https://localhost:8090",
    clientId: "<your-client-id>",
    applicationId: "<your-application-id>",
    afterSignInUrl: "myapp://callback",
    afterSignOutUrl: "myapp://",
    scopes: ["openid", "profile", "email"]
)
```

## Properties

| Property | Type | Required | Default | Description |
| :--- | :--- | :---: | :--- | :--- |
| `baseUrl` | `String` | ✅ | — | Base URL of your Thunder instance (must use HTTPS) |
| `clientId` | `String?` | ❌ | `nil` | Client ID of your Thunder application. Required for redirect-based sign-in |
| `applicationId` | `String?` | ❌ | `nil` | Application ID of your Thunder application. Required for embedded sign-in |
| `afterSignInUrl` | `String?` | ❌ | `nil` | URI to redirect to after a successful sign-in (for example, `myapp://callback`) |
| `afterSignOutUrl` | `String?` | ❌ | `nil` | URI to navigate to after sign-out |
| `signInUrl` | `String?` | ❌ | `nil` | Custom sign-in page URL. Overrides the default Thunder sign-in URL |
| `signUpUrl` | `String?` | ❌ | `nil` | Custom sign-up page URL |
| `scopes` | `[String]` | ❌ | `["openid"]` | OAuth2 scopes to request during sign-in |
| `clientSecret` | `String?` | ❌ | `nil` | Client secret. Only required for confidential clients |
| `signInOptions` | `[String: Any]` | ❌ | `[:]` | Additional parameters appended to the authorization request |
| `signOutOptions` | `[String: Any]` | ❌ | `[:]` | Additional parameters for the sign-out request |
| `signUpOptions` | `[String: Any]` | ❌ | `[:]` | Additional parameters for the sign-up request |
| `organizationHandle` | `String?` | ❌ | `nil` | Organization handle to scope the session at initialization |
| `tokenValidation` | `TokenValidationConfig` | ❌ | See below | Token validation settings |
| `storage` | `StorageAdapter?` | ❌ | `nil` | Custom storage adapter. Defaults to `KeychainStorageAdapter` if not provided |
| `instanceId` | `Int?` | ❌ | `nil` | Optional identifier for multi-instance SDK setups |

## TokenValidationConfig

Controls how the SDK validates ID tokens received from Thunder.

```swift
let config = ThunderConfig(
    baseUrl: "https://localhost:8090",
    tokenValidation: TokenValidationConfig(
        validate: true,
        validateIssuer: true,
        clockTolerance: 60
    )
)
```

| Property | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `validate` | `Bool` | `true` | Whether to validate ID tokens. Set to `false` only in development |
| `validateIssuer` | `Bool` | `true` | Whether to validate the `iss` claim against `baseUrl` |
| `clockTolerance` | `Int` | `0` | Allowed clock skew in seconds when checking `exp` and `iat` claims |

## StorageAdapter

`StorageAdapter` is a protocol for custom token storage. By default, the SDK uses `KeychainStorageAdapter` which stores tokens securely in the iOS Keychain. Implement this protocol if you need a custom storage backend.

```swift
public protocol StorageAdapter {
    func get(key: String) async throws -> String?
    func set(key: String, value: String) async throws
    func remove(key: String) async throws
}
```
