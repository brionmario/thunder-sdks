# `ThunderClient`

`ThunderClient` is the main entry point for all Thunder authentication and user management operations in the Swift SDK. Methods use Swift Concurrency (`async throws`), except `isLoading()`, `buildSignInURL()`, `decodeJwtToken(_:)`, `getConfiguration()`, and `clearSession()`.

## Usage

```swift
let client = ThunderClient()
try await client.initialize(config: ThunderConfig(
    baseUrl: "https://localhost:8090",
    clientId: "<your-client-id>",
    applicationId: "<your-application-id>"
))
```

---

## Lifecycle

### `initialize`

```swift
public func initialize(config: ThunderConfig, storage: StorageAdapter? = nil) async throws -> Bool
```

Initializes the SDK with the provided configuration. Must be called once before any other method.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `config` | `ThunderConfig` | SDK configuration |
| `storage` | `StorageAdapter?` | Custom storage adapter. Defaults to `KeychainStorageAdapter` |

**Returns:** `true` when initialization succeeds.

**Throws:** `IAMError(.alreadyInitialized)` if called more than once.

---

### `reInitialize`

```swift
public func reInitialize(baseUrl: String? = nil, clientId: String? = nil) async throws -> Bool
```

Updates one or more configuration fields and re-initializes the SDK.

**Returns:** `true` when re-initialization succeeds.

---

### `getConfiguration`

```swift
public func getConfiguration() throws -> ThunderConfig
```

Returns the current configuration. **Synchronous.**

**Throws:** `IAMError(.sdkNotInitialized)` if the SDK has not been initialized.

## Authentication

### `signIn`

```swift
public func signIn(
    payload: EmbeddedSignInPayload,
    request: EmbeddedFlowRequestConfig,
    sessionId: String? = nil
) async throws -> EmbeddedFlowResponse
```

Drives an app-native sign-in via the Flow Execution API. Call in a loop until `flowStatus == .complete`.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `payload` | `EmbeddedSignInPayload` | Flow payload. Set `actionId = "init"` to start a new flow |
| `request` | `EmbeddedFlowRequestConfig` | Flow request configuration including `applicationId` |
| `sessionId` | `String?` | Optional session identifier for multi-session support |

**Returns:** `EmbeddedFlowResponse` containing `flowId`, `flowStatus`, and step data.

**Example:**

```swift
// Step 1: Initiate
let init = try await client.signIn(
    payload: EmbeddedSignInPayload(actionId: "init"),
    request: EmbeddedFlowRequestConfig(applicationId: "your-app-id")
)

// Step 2: Submit credentials
let result = try await client.signIn(
    payload: EmbeddedSignInPayload(
        flowId: init.flowId,
        actionId: "basic_auth",
        inputs: ["username": "user@example.com", "password": "secret"]
    ),
    request: EmbeddedFlowRequestConfig(applicationId: "your-app-id")
)

if result.flowStatus == .complete {
    let user = try await client.getUser()
}
```

### `buildSignInURL`

```swift
public func buildSignInURL(options: SignInOptions? = nil) throws -> URL
```

Builds and returns the OAuth2 authorization `URL` for redirect-based sign-in. Pass this URL to `ASWebAuthenticationSession`. **Synchronous.**

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `options` | `SignInOptions?` | Optional parameters such as `prompt`, `loginHint`, and `fidp` |

**Returns:** The authorization `URL` with PKCE challenge included.

**Throws:** `IAMError(.invalidConfiguration)` if `clientId` is not set.

---

### `handleRedirectCallback`

```swift
public func handleRedirectCallback(url: String) async throws -> User
```

Exchanges the authorization code in the callback URL for tokens and returns the signed-in user.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `url` | `String` | The full callback URL from `ASWebAuthenticationSession` or the URL scheme handler |

**Returns:** The authenticated `User` object.

---

### `signOut`

```swift
public func signOut(options: SignOutOptions? = nil, sessionId: String? = nil) async throws -> String
```

Revokes the refresh token server-side and clears all local session state.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `options` | `SignOutOptions?` | Optional sign-out parameters such as `idTokenHint` |
| `sessionId` | `String?` | Optional session identifier |

**Returns:** The `afterSignOutUrl` from configuration, or `"/"` if not set.

---

### `isSignedIn`

```swift
public func isSignedIn(sessionId: String? = nil) async throws -> Bool
```

Returns `true` if a valid access token is stored in the current session.

---

### `isLoading`

```swift
public func isLoading() -> Bool
```

Returns `true` while the SDK is mid-initialization or mid-token-refresh. **Synchronous** — safe to call from any thread.

## Token & Session

### `getAccessToken`

```swift
public func getAccessToken(sessionId: String? = nil) async throws -> String
```

Returns the current access token, transparently refreshing it if it is near expiry.

**Throws:** `IAMError(.sessionExpired)` if no valid session exists and the token cannot be refreshed.

---

### `decodeJwtToken`

```swift
public func decodeJwtToken<R>(_ token: String) throws -> R
```

Decodes the payload of a JWT without verifying the signature. **Synchronous.**

:::warning
Do not use this method for authorization decisions. Always validate tokens server-side.
:::

---

### `exchangeToken`

```swift
public func exchangeToken(
    config: TokenExchangeRequestConfig,
    sessionId: String? = nil
) async throws -> TokenResponse
```

Performs an RFC 8693 token exchange. Primarily used for organization switching.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `config` | `TokenExchangeRequestConfig` | Exchange parameters including `subjectToken` and `audience` |

**Returns:** A new `TokenResponse` scoped to the target audience.

---

### `clearSession`

```swift
public func clearSession(sessionId: String? = nil)
```

Clears all local session state without server-side token revocation. **Synchronous.** Use `signOut()` for a full sign-out.

## User & Profile

### `getUser`

```swift
public func getUser(options: [String: Any]? = nil) async throws -> User
```

Fetches verified user claims from the `/oauth2/userinfo` endpoint. The result is cached; call `clearSession()` to invalidate the cache.

**Returns:** A `User` struct with `sub`, `username`, `email`, `displayName`, and `profilePicture`.

---

### `getUserProfile`

```swift
public func getUserProfile(options: [String: Any]? = nil) async throws -> UserProfile
```

Fetches the full SCIM 2.0 user profile from `/scim2/Me`.

**Returns:** A `UserProfile` struct with `id` and a `claims` dictionary.

---

### `updateUserProfile`

```swift
public func updateUserProfile(
    payload: [String: Any],
    userId: String? = nil
) async throws -> User
```

Updates user profile attributes.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `payload` | `[String: Any]` | Dictionary of profile attributes to update |
| `userId` | `String?` | Target user ID. Defaults to the currently signed-in user |

**Returns:** The updated `User` struct.

## Data Types

### `EmbeddedSignInPayload`

| Property | Type | Description |
| :--- | :--- | :--- |
| `flowId` | `String?` | Flow identifier from the previous response. `nil` to start a new flow |
| `actionId` | `String` | Action to execute (for example, `"init"`, `"basic_auth"`) |
| `inputs` | `[String: String]` | Input values for the current step |

### `EmbeddedFlowResponse`

| Property | Type | Description |
| :--- | :--- | :--- |
| `flowId` | `String?` | Flow identifier to pass in the next request |
| `flowStatus` | `FlowStatus` | `.promptOnly`, `.complete`, or `.error` |
| `stepId` | `String?` | Current step identifier |
| `data` | `FlowStepData?` | Available actions and input fields for the current step |
| `assertion` | `String?` | Authorization code or token when `flowStatus == .complete` |
| `failureReason` | `String?` | Error description when `flowStatus == .error` |

### `User`

| Property | Type | Description |
| :--- | :--- | :--- |
| `sub` | `String` | Subject identifier (unique user ID) |
| `username` | `String?` | Username |
| `email` | `String?` | Email address |
| `displayName` | `String?` | Display name |
| `profilePicture` | `String?` | Profile picture URL |
| `isNewUser` | `Bool?` | Whether this is the user's first sign-in |
| `claims` | `[String: Any]?` | Additional claims from the ID token |

### `Organization`

| Property | Type | Description |
| :--- | :--- | :--- |
| `id` | `String` | Organization identifier |
| `name` | `String` | Organization display name |
| `handle` | `String?` | URL-friendly organization handle |

## Error Handling

All `async throws` methods throw `IAMError` on failure.

```swift
do {
    let user = try await client.getUser()
} catch let error as IAMError {
    switch error.code {
    case .sessionExpired:
        // prompt re-authentication
    case .networkError:
        // show connectivity error
    case .authenticationFailed:
        // show wrong credentials message
    default:
        // generic error handling
    }
}
```

### `IAMErrorCode` Values

| Code | Description |
| :--- | :--- |
| `.sdkNotInitialized` | A method was called before `initialize()` |
| `.alreadyInitialized` | `initialize()` was called more than once |
| `.invalidConfiguration` | A required configuration value is missing or invalid |
| `.authenticationFailed` | Sign-in credentials were rejected |
| `.sessionExpired` | The session has expired and cannot be refreshed |
| `.mfaRequired` | Multi-factor authentication is required to continue |
| `.userAlreadyExists` | Sign-up attempted for an already-registered email |
| `.invalidGrant` | The authorization code or refresh token is invalid |
| `.invalidInput` | An input parameter is malformed |
| `.networkError` | A network request failed |
| `.tokenValidationFailed` | The received ID token failed validation |
| `.unknown` | An unexpected error occurred |
