# `ThunderClient`

`ThunderClient` is the core SDK client for all Thunder authentication and user management operations. It communicates with the native iOS and Android SDKs via a platform channel. All methods return `Future`s, except `isLoading()` and `clearSession()`.

Access `ThunderClient` via `ThunderProvider.of(context).client`, or instantiate it directly for testing.

## Usage

```dart
final client = ThunderClient();
await client.initialize(ThunderConfig(
  baseUrl: 'https://localhost:8090',
  clientId: '<your-client-id>',
  applicationId: '<your-application-id>',
));
```

---

## Lifecycle

### `initialize`

```dart
Future<bool> initialize(ThunderConfig config)
```

Initializes the SDK with the provided configuration. Must be called once before any other method. `ThunderProvider` calls this automatically at startup.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `config` | `ThunderConfig` | SDK configuration |

**Returns:** `true` when initialization succeeds.

**Throws:** `IAMException(IAMErrorCode.alreadyInitialized)` if called more than once.

---

### `reInitialize`

```dart
Future<bool> reInitialize({String? baseUrl, String? clientId})
```

Updates one or more configuration fields and re-initializes the SDK.

**Returns:** `true` when re-initialization succeeds.

## Authentication

### `signIn`

```dart
Future<EmbeddedFlowResponse> signIn({
  required EmbeddedSignInPayload payload,
  required EmbeddedFlowRequestConfig request,
  String? sessionId,
})
```

Drives an app-native sign-in via the Flow Execution API. Call in a loop until `flowStatus == FlowStatus.complete`.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `payload` | `EmbeddedSignInPayload` | Flow payload. Set `actionId = 'init'` to start a new flow |
| `request` | `EmbeddedFlowRequestConfig` | Flow request configuration including `applicationId` |
| `sessionId` | `String?` | Optional session identifier for multi-session support |

**Returns:** `EmbeddedFlowResponse` containing `flowId`, `flowStatus`, and step data.

**Example:**

```dart
// Step 1: Initiate
final init = await client.signIn(
  payload: EmbeddedSignInPayload(actionId: 'init'),
  request: EmbeddedFlowRequestConfig(applicationId: 'your-app-id'),
);

// Step 2: Submit credentials
final result = await client.signIn(
  payload: EmbeddedSignInPayload(
    flowId: init.flowId,
    actionId: 'basic_auth',
    inputs: {'username': 'user@example.com', 'password': 'secret'},
  ),
  request: EmbeddedFlowRequestConfig(applicationId: 'your-app-id'),
);

if (result.flowStatus == FlowStatus.complete) {
  await thunder.refresh();
}
```

### `buildSignInUrl`

```dart
Future<String> buildSignInUrl({SignInOptions? options})
```

Returns the OAuth2 authorization URL for redirect-based sign-in. Open this URL with `url_launcher` or a custom tab.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `options` | `SignInOptions?` | Optional parameters such as `prompt`, `loginHint`, and `fidp` |

**Returns:** The authorization URL string with PKCE challenge included.

---

### `handleRedirectCallback`

```dart
Future<User> handleRedirectCallback(String url)
```

Exchanges the authorization code in the callback URL for tokens and returns the signed-in user.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `url` | `String` | The full callback URL received from the deep link handler |

**Returns:** The authenticated `User` object.

---

### `signOut`

```dart
Future<String> signOut({SignOutOptions? options, String? sessionId})
```

Revokes the refresh token server-side and clears all local session state.

**Returns:** The `afterSignOutUrl` from configuration, or `'/'` if not set.

---

### `isSignedIn`

```dart
Future<bool> isSignedIn({String? sessionId})
```

Returns `true` if a valid access token is stored in the current session.

---

### `isLoading`

```dart
bool isLoading()
```

Returns `true` while the SDK is mid-initialization or mid-token-refresh. **Synchronous** — safe to call from `build()`.

  EmbeddedSignInPayload? payload,

Drives an app-native user registration via the Flow Execution API. Follows the same loop pattern as `signIn()`.

## Token & Session

### `getAccessToken`

```dart
Future<String> getAccessToken({String? sessionId})
```

Returns the current access token, transparently refreshing it if near expiry.

**Throws:** `IAMException(IAMErrorCode.sessionExpired)` if no valid session exists.

---

### `decodeJwtToken`

```dart
Future<Map<String, dynamic>> decodeJwtToken(String token)
```

Decodes the payload of a JWT without verifying the signature.

:::warning
Do not use this method for authorization decisions. Always validate tokens server-side.
:::

---

### `exchangeToken`

```dart
Future<TokenResponse> exchangeToken(TokenExchangeRequestConfig config, {String? sessionId})
```

Performs an RFC 8693 token exchange. Primarily used for organization switching.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `config` | `TokenExchangeRequestConfig` | Exchange parameters including `subjectToken` and `audience` |

**Returns:** A new `TokenResponse` scoped to the target audience.

---

### `clearSession`

```dart
void clearSession({String? sessionId})
```

Clears local session state without server-side token revocation. **Synchronous.** Use `signOut()` for a full sign-out.

## Data Types

### `EmbeddedSignInPayload`

| Property | Type | Description |
| :--- | :--- | :--- |
| `flowId` | `String?` | Flow identifier from the previous response. `null` to start a new flow |
| `actionId` | `String` | Action to execute (for example, `'init'`, `'basic_auth'`) |
| `inputs` | `Map<String, String>` | Input values for the current step |

### `EmbeddedFlowResponse`

| Property | Type | Description |
| :--- | :--- | :--- |
| `flowId` | `String?` | Flow identifier to pass in the next request |
| `flowStatus` | `FlowStatus` | `FlowStatus.promptOnly`, `.complete`, or `.error` |
| `stepId` | `String?` | Current step identifier |
| `data` | `FlowStepData?` | Available actions and input fields for the current step |
| `assertion` | `String?` | Authorization code or token when `flowStatus == complete` |
| `failureReason` | `String?` | Error description when `flowStatus == error` |

### `User`

| Property | Type | Description |
| :--- | :--- | :--- |
| `sub` | `String` | Subject identifier (unique user ID) |
| `username` | `String?` | Username |
| `email` | `String?` | Email address |
| `displayName` | `String?` | Display name |
| `profilePicture` | `String?` | Profile picture URL |
| `isNewUser` | `bool?` | Whether this is the user's first sign-in |
| `claims` | `Map<String, dynamic>?` | Additional claims from the ID token |

### `Organization`

| Property | Type | Description |
| :--- | :--- | :--- |
| `id` | `String` | Organization identifier |
| `name` | `String` | Organization display name |
| `handle` | `String?` | URL-friendly organization handle |

## Error Handling

All async methods throw `IAMException` on failure.

```dart
try {
  await thunder.client.signIn(...);
} on IAMException catch (e) {
  switch (e.code) {
    case IAMErrorCode.authenticationFailed:
      // show wrong credentials message
    case IAMErrorCode.sessionExpired:
      // prompt re-authentication
    case IAMErrorCode.networkError:
      // show connectivity error
    default:
      // generic error handling
  }
}
```

### `IAMErrorCode` Values

| Code | Description |
| :--- | :--- |
| `IAMErrorCode.sdkNotInitialized` | A method was called before `initialize()` |
| `IAMErrorCode.alreadyInitialized` | `initialize()` was called more than once |
| `IAMErrorCode.invalidConfiguration` | A required configuration value is missing or invalid |
| `IAMErrorCode.authenticationFailed` | Sign-in credentials were rejected |
| `IAMErrorCode.sessionExpired` | The session has expired and cannot be refreshed |
| `IAMErrorCode.mfaRequired` | Multi-factor authentication is required to continue |
| `IAMErrorCode.userAlreadyExists` | Sign-up attempted for an already-registered email |
| `IAMErrorCode.invalidGrant` | The authorization code or refresh token is invalid |
| `IAMErrorCode.invalidInput` | An input parameter is malformed |
| `IAMErrorCode.networkError` | A network request failed |
| `IAMErrorCode.tokenValidationFailed` | The received ID token failed validation |
| `IAMErrorCode.unknown` | An unexpected error occurred |
